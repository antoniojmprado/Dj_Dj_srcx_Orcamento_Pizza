Sua atribuição acontece dentro de algum fetch, banco de dados ou evento?



Sim. Minha declaração foi feita dentro de um fetch.
Aí está o "pulo do gato"! O fetch é assíncrono, o que significa que o JavaScript inicia o pedido e continua executando o restante do código sem esperar a resposta.
Quando você tenta acessar a variável logo abaixo do fetch, o servidor ainda não respondeu, por isso ela continua null ou undefined.
Como resolver isso?
Você tem duas opções principais dependendo de como seu código está estruturado:
1. Usar async/await (A forma mais moderna e limpa)
Transforme a função que contém o código em async. Isso permite que você use o await, forçando o JavaScript a "pausar" até que o valor seja atribuído à global.
javascript
let minhaGlobal = null;

async function inicializar() {
    const resposta = await fetch('https://api.exemplo.com');
    const dados = await resposta.json();
    
    minhaGlobal = dados.valor; // Agora a global está preenchida
    
    proximaEtapa(); // Chame a função que depende do valor AQUI dentro
}

function proximaEtapa() {
    console.log("Agora o valor existe:", minhaGlobal);
}

inicializar();
Use o código com cuidado.

2. Encadear a lógica dentro do .then()
Se não quiser usar async/await, toda a lógica que depende dessa variável precisa ser disparada de dentro do último .then().
javascript
let minhaGlobal = null;

fetch('https://api.exemplo.com')
  .then(res => res.json())
  .then(dados => {
    minhaGlobal = dados.valor;
    // Qualquer código que use 'minhaGlobal' deve ser chamado aqui:
    document.getElementById("resultado").innerText = minhaGlobal;
  });
Use o código com cuidado.

Por que isso acontece?
Imagine o fetch como pedir uma pizza. Você faz o pedido (chama o fetch) e continua assistindo TV. Se você tentar comer (usar a variável) um segundo após o pedido, a pizza ainda não chegou. Você só pode comer depois que o entregador tocar a campainha (o .then ou o await).
Dica extra: Para evitar erros, você pode inicializar sua variável como null e sempre verificar if (minhaGlobal) { ... } antes de usar.
Você está tentando usar esse valor para preencher algo na tela ou para fazer outro cálculo logo em seguida?



Para fazer uma comparação e imprimir se ela é maior que outra varíável.
Para que essa comparação funcione, ela obrigatoriamente deve acontecer após o fetch ser concluído. Se você fizer a comparação fora (no fluxo principal), o JavaScript vai comparar null com o número e o resultado será falso ou imprevisível.
Aqui está o jeito certo de estruturar isso usando async/await, que é o mais legível:
javascript
let minhaGlobal = null; // Declarada no topo
let outraVariavel = 50;

async function buscarEComparar() {
  try {
    const resposta = await fetch('https://api.exemplo.com');
    const dados = await resposta.json();
    
    // Atribui o valor à global
    minhaGlobal = dados.preco; 

    // A COMPARAÇÃO DEVE ESTAR AQUI (ou ser chamada daqui)
    realizarComparacao();

  } catch (erro) {
    console.error("Erro na requisição:", erro);
  }
}

function realizarComparacao() {
  if (minhaGlobal > outraVariavel) {
    console.log(`Sucesso: ${minhaGlobal} é maior que ${outraVariavel}`);
  } else {
    console.log("É menor ou igual.");
  }
}

buscarEComparar();
Use o código com cuidado.

Por que fazer assim?
Garantia de Dado: O await garante que minhaGlobal tenha o valor da API antes de entrar no if.
Organização: Ao criar a função realizarComparacao, você mantém o código limpo, mas só a "gatilha" quando os dados chegam.
Segurança: Se a API falhar, o catch evita que seu código quebre ao tentar comparar um valor inexistente.
Dica de ouro: No JavaScript moderno, para evitar que o código tente comparar null com um número por acidente, você pode usar um valor padrão na declaração: let minhaGlobal = 0;.
